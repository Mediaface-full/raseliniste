## Klíčové funkční změny

### Real-time sync progress

`User.contactsSyncProgress` JSON drží stav sync během běhu:
```ts
{ provider: "icloud"|"google",
  stage: "discovery"|"listing"|"fetching"|"saving"|"merging"|"done"|"error",
  current: N, total: N, mergedClusters: N,
  startedAt: ISO, message?: string, error?: string }
```

- Lib `src/lib/contacts-sync-progress.ts` (start/update/finish/get/clear)
- iCloud + Google sync funkce volají `updateSyncProgress` v krocích
- Endpoint `GET /api/contacts/sync-progress` pro polling
- Frontend `ContactsTable.tsx` polluje à 2s pokud `syncingIcloud || syncingGoogle`, zobrazí Loader + `current/total` + progress bar + message
- Po `done` server-side 30s auto-clear, frontend 5s držet banner

### Custom AI prompt pro projektovou analýzu

**Default** (žádný custom prompt v projektu): Stage 2 vrátí strukturovaný JSON (transcript, summary, key_themes, thoughts, open_questions, sentiment, intensity_signals + brief má glossary/actors/decision_history). UI defaults beze změny.

**S custom prompt** (Petr napíše do `studnaStandardPrompt` nebo `studnaBriefPrompt`): JSON má JEN 2 pole — `transcript` (prázdné) + `customExtract` (volný markdown řízený Petrovým pokynem). UI zobrazí **mint box „🎯 Vlastní extrakt"** s markdown render.

**Příklad:**
```
Petr v studnaStandardPrompt projektu napíše:
"Vytáhni seznam úkolů co z toho vyplynou: kdo, co, dokdy. Bullet list."

Stage 2 vrátí:
{
  "transcript": "",
  "customExtract": "## Úkoly\n- [ ] Karel: poslat smlouvu do pátku\n- [ ] Petr: ..."
}
```

**Pozor:** Stará UI měla matoucí pole `extractionPrompt` (label „Vlastní AI prompt") které se v žádném AI volání nepoužívalo (dead code). Odstraněno v commit `c27cabc`, DB field zachován pro budoucnost.

### Velký toolbar (podle briefu 5.5)

Hero (kompaktní):
- Title + count + dirty
- Status pily iCloud / Google
- Sync iCloud / Sync Google tlačítka

Toolbar (samostatný blok):
- **Obnovit** (reload z DB) + **Zahodit** (clear dirty)
- Search napříč `displayName/firstName/lastName/company/phones/emails/note/groups`
- Page size + Validační filtr
- **Uložit (iCloud)** = save DB + push iCloud per-row
- **Uložit + Google** = save + push iCloud + obousměrný Google sync
- **+ Nový kontakt** (prompt → POST /api/contacts)
- **+ Nová skupina** (prompt → POST /api/contacts/groups)

### Delete řádek (DB + iCloud + Google)

Trash ikona v posledním sloupci tabulky:
- Auto-backup před delete
- Best-effort `deleteVCard` z iCloudu (pokud má `icloudHref`)
- Best-effort `people.deleteContact` z Google (pokud má `googleResourceName`)
- Hard DELETE z DB (cascade phones/emails)
- Toast hlásí výsledek: `iCloud ✓ · Google ✓ · z DB ✓`

Endpoint: `POST /api/contacts/:id/delete-everywhere`

### `&#13;` posílený cleanup

Apple iCloud zkomoleně posílá `&#13;` (HTML entity) v emailech/telefonech/jménech/adresách. Cleanup endpoint:

- `decodeEntities` zvládá numeric entities **i bez `;`** (zkomolený output)
- Dvojí průchod (`&#13;` po dekódování zbytky)
- Speciální handler `decodeAddress` pro adresy zachovává `\n` mezi řádky ulice/město/země
- Mažou se prázdné kontakty (bez čitelného jména + bez contact info + bez overlay flags)

UI: **🧹 Vyčistit entity + prázdné** tlačítko v Nástrojích banneru. **💣 Nuclear reset** smaže non-overlay a sync iCloudu znovu (pro extrémní bordel).

### Filter „Klienti" (a další skupiny)

Klik na chip skupiny v UI nastaví search → backend `/api/contacts/tabulka` teď hledá taky v `groups: { has: q }` array (předtím jen displayName/firstName/lastName/company/phones/emails/note).
