### Bug fixy a cleanup tooly

#### Calendar duplicates / TZ bugs
- **Diagnostic + cleanup endpoint** `/api/diagnose/calendar-cleanup` — najde same-source duplikáty (artefakt pgvector swapu z 04-30) a smaže
- **Calendar all-day timestamp normalize** — fix multi-day spans bug ve **třech vrstvách**:
  1. **Server**: `parseEventDate` v google-calendar.ts používá `Date.UTC()` (deterministické, nezávislé na server TZ)
  2. **Klient**: WeekView allDaySpans porovnává **kalendářní dny v Praha TZ** (přes `toLocaleDateString sv-SE`), offset −12h pro `endAdjusted`
  3. **DB cleanup**: `/api/diagnose/calendar-allday-fix` normalizuje historicky uložené posunuté timestampy → UI tlačítko v `/settings/calendar-cleanup`
- **Boundary `gte` → `gt`** napříč endpointy (`/api/day/[date]/index.ts`, `lib/briefing.ts`, `pages/day/[date].astro`) — sobotní all-day už nepadá do nedělního pohledu
- **Google sync sweep** — předtím se delete chytaly jen přes cancelled status (showDeleted=true). Teď seenIds + sweep updateMany označí jako `deletedRemotely` všechny rows v window které nejsou v API response. Safety: jen pokud paging dokončil bez errorů. Vyřeší smazané schůzky co dříve „svítily" v UI.

#### OOO / Dovolená

- **`/api/calendar/away` přestal posílat outOfOffice flag** — Petr má vlastní rules engine pro auto-decline. Fantastical zobrazí celodenní badge místo gigantického modrého bloku.
