## 🟢 CO SE TENTO DEN UDĚLALO

### Kalendář — kompletní redesign

**Tři pohledy s sjednocenou estetikou:**

- **Den** (`/day/<datum>`) — vertikální timeline, mobilní, time-blindness friendly
  - Barvy podle source, sloupce při overlapu, long-event-as-background
  - Mezera 2 px mezi bloky, čas nahoře / název pod
  - Now čára v terakotě, scroll-to-now, source badge podmíněně
  - Length badge "3 H / 6 H" pro long events
  - Sliding window: gridStart = first event − 1h
- **Týden** (`/calendar/tyden`) — desktop
  - 7 sloupců (Po-Ne), 6:00-23:00, 56 px/h
  - Now čára napříč všemi sloupci, badge VLEVO v gutteru (ne vpravo)
  - Multi-day all-day spanning (greedy row assignment, max 2 + expand)
  - 3 default + custom rituály + výročí
  - Interpretační lišta (4-6 faktických vět) — žádné hodnocení
- **Měsíc** (`/calendar/mesic`) — desktop ORIENTAČNÍ
  - Po redesignu: hustotní podbarvení (5 stupňů), velké eventy jako TEXT v buňce, rituály ✨ v rohu
  - Aktuální den + týden zvýrazněné
  - **Hover tooltip** (200 ms fade) s plným seznamem dne
  - Klik na buňku → /day/<datum>
  - „Květen 2026" v nominativu

**Společné:**
- **Topbar quick buttons** Den / Týden / Měsíc na všech stránkách
- **Přepínač pohledů** v hlavičce každého (i ve fullscreen módu)
- **Tlačítko Tisk** — `window.print()` + `@media print` v global.css (A4 landscape, schová UI)

  - Schema `CustomRitual` (title, description, daysOfWeek[], startHour, startMinute, durationMin, active)
  - Opakování: presety (každý den / Po-Pá / víkend / vybrané dny — checkboxy)

### Zdraví dat — bug fixy + cleanup tooly

- **Calendar duplicates cleanup** — `/settings/calendar-cleanup`. Najde stejné `(source, externalId)` rows (artefakt pgvector image swapu z 04-30) a smaže starší. UI tool s DRY RUN.
- **Calendar all-day timestamp normalize** — fix multi-day spans bug:
  - **Root cause**: `parseEventDate` v `google-calendar.ts` parsoval `"2026-05-09T00:00:00"` jako server-local time. Server v Praha TZ → posun timestampu o 22 h. V DB jsou historicky uložené eventy s nepřesnými timestampy.
  - **Fix v 3 vrstvách**: server parse → `Date.UTC()` deterministicky / klient offset → −12h (předtím −4h nestačil) / DB cleanup → endpoint `POST /api/diagnose/calendar-allday-fix` s UI tlačítkem „Normalizovat all-day eventy"
- **Boundary `gte` → `gt`** napříč více endpointy (`/api/day/[date]/index.ts`, `lib/briefing.ts`, `pages/api/day/[date]/index.ts`) — sobotní all-day už nepadá do nedělního pohledu/briefingu
