### G) Audio upload mobile fix (`28141c1` `064fc58`)

Petr nahlásil že na iPhone nahrávání audio do úkolů „neudělá nic".

Root cause: tlačítko „Nahrát soubor" v `/ukoly` přesměrovávalo na
`/ozvena?upload=1`, kde DiktatRecorder po 250 ms volal
`fileInputRef.current.click()`. Mobile Safari blokuje programmatic click
na file input z setTimeout — anti-popup ochrana. Musí být přímá reakce
na user gesture v aktuální stránce.

**Fix:**
- UkolyList: skrytý `<input type=file>` + tlačítko ho clickne v onClick
  handleru. Upload přes POST `/api/ukoly/audio` → redirect na
  `/ukoly/audio/<id>/review` (POZOR: ne bez `/review` suffixu — to byl
  follow-up bug v `097ac19`).
- DenikList: stejný pattern, redirect na `/denik/<entryId>`.
- DiktatRecorder: vyhozen problematický `setTimeout(click, 250)` —
  primary cesty teď nepřesměrovávají sem.

### H) AI extract — odolnost + počet úkolů (`5bbfc80` `cfe3e43` `097ac19` `b0f01bf`)

Petr nahlásil 29min audio, AI vrátila jen 5 úkolů. Plus některé batche
selhávaly s JSON parse errorem (`Unexpected non-whitespace character`).

**Fixy v pořadí:**

1. `tolerantParseTasks()` v `process-task-audio.ts` — balanced
   brace/bracket scan najde první kompletní `{…}` nebo `[…]`, sní
   přebytky, root array wrapuje do `{ tasks: [...] }`. Respektuje
   stringy + escape characters. Před tím standard `JSON.parse` padal
   když AI vrátila root array nebo přidala druhý objekt za první.

2. Token limit `maxOutputTokens` 12 000 → 30 000 → 60 000 (`097ac19`,
   `b0f01bf`). Pro 80+ úkolů s `notes` (max 1500 znaků) limit doběhl.

3. `thinkingConfig.thinkingBudget = 4096` (`b0f01bf`) — Gemini 2.5 Pro
   reasoning si může vzít tisíce tokenů z output budget na thinking.
   Explicit nízký budget šetří output pro JSON.

4. **Prompt — žádný strop** (`cfe3e43`, `b0f01bf`):
   - Smazáno „krátká salva 30 s – 2 min" (AI to brala jako limit)
   - Tabulka očekávaných počtů: 30 min → 40-100+, 60 min → 80-200+
   - Explicit „NIKDY nekonsoliduj přes hranice témat, NIKDY nezahazuj"
   - „Lepší 100 v review odškrtnu než 5 musím znova diktovat"
   - Safety net: při pochybnosti vytáhni s priority=low

**Verified výsledek**: 29min audio, dříve 5 úkolů, po fix **36 úkolů**.

## Klíčové learnings (pro budoucí session)

1. **SQL NULL + Prisma NOT** — `NOT { field: "X" }` na nullable poli
   skrývá řádky kde `field IS NULL`. Vždy explicit `OR [{ field: null }, ...]`
   nebo `not: { equals: "X" }` (Prisma 5+ pak NULL zahrnuje).

2. **Mobile Safari + programmatic file input click** — vždy click jen
   z user gesture v aktuální stránce. Žádný setTimeout, žádný redirect
   + auto-click. Direct picker je jediná spolehlivá cesta.

3. **Gemini 2.5 Pro thinking** — reasoning models si bez explicit limitu
   můžou vzít tisíce tokenů na thinking z naší output kvóty. Pro
   strukturovaný JSON output dávat `thinkingConfig.thinkingBudget = 4096`
   (nebo 0 pro maximum output).

4. **Astro 6 standalone log** — defaultně neloguje 400/401 responses
   (jen errors). Pro debug uživatelských stížností přidat explicit
   `console.log/warn/error` do endpointu.

5. **Google Calendar event description** — plain text, ne markdown.
   `**bold**` se zobrazí doslova.

6. **Petr nesnáší fire-and-forget bez kontextu** — pokud spinner běží,
   musí mít context: kterou fázi, jak dlouho, co dělá. Tichý fire-
   and-forget je OK jen pro krátké (<5s) operace.

7. **Prompt limity AI** — když řeknu „krátká salva 30 s – 2 min" v
   promptu, AI to interpretuje jako instrukci „udělej málo úkolů" i pro
   30 min audio. Vždy explicit horní hranice nebo „žádný strop".
