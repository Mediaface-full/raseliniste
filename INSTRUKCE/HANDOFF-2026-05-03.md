## 📂 KLÍČOVÉ KOMPONENTY (rychlá orientace)

### Todoist plně obousměrný sync

**Z Rašeliniště → Todoist (synchronně, user-action):**

| Akce | Todoist | Endpoint |
|---|---|---|
| Audio diktát commit | Auto-push všech tasků (parent+children) | `audio/[batchId]/commit.ts` |
| Manual create v `/ukoly` | Auto-push | `POST /api/ukoly` |
| Edit title/notes/dueAt/labels/priority | `updateTask` | `PATCH /api/ukoly/:id` |
| Mark done | `closeTask` | `PATCH /api/ukoly/:id` |
| Reopen | `reopenTask` | `PATCH /api/ukoly/:id` |
| Delete | `deleteTask` (idempotent 404) | `DELETE /api/ukoly/:id` |
| VIP firewall submit | `createTask` v projektu vip | `POST /api/call-log/submit` |

**Z Todoistu → Rašeliniště (cron `todoist-sync` každých 5 min):**

1. **Items pull** — incremental přes `User.todoistSyncToken`
   - Nový task v Todoist appce → vytvoří `Task` se `source=todoist_pull`
   - Update title/notes/due/labels/priority (Todoist je zdroj pravdy)
   - Reopen → čistí `completedAt` (commit `98b40be`)
2. **Projects + labels mirror** — `TodoistProjectMirror`, `TodoistLabelMirror`
3. **Reconcile pass** (klíčové, Sync API completed nevrací):
   - `getTask(token, todoistTaskId)`
   - `null` → hard-deleted v Todoistu → close lokálně
   - `is_completed: true` → odškrtnut → close lokálně
   - `is_completed: false` + lokálně done → reopen lokálně

### VIP firewall

- **Schema:** `Contact.callLogToken @unique` (24 znaků base64url, 144 bit entropie)
- **Lib:** `src/lib/call-log-token.ts` (resolveCallLogToken s defense-in-depth `isVip=true`), `src/lib/vip-missions.ts` (loadVipMissions s STRIKT contactId match)
- **Frontend:** `/call-log?t=<token>` (zadávací + výpis), `/call-log/thanks?t=<token>` (potvrzení + výpis)
- **Backfill:** `/contacts` toolbar tlačítko „VIP tokeny"
- **Kopírování linku:** ikona řetízku v seznamu kopíruje TOKEN URL pro VIP (commit `afcba79`)
- **Bezpečnost:** cross-VIP průsak fyzicky nemožný (audit 8 míst)
