Cron nastavení na NASu

Kompletní výpis všech automatických úloh, které musí běžet na Synology DSM Task Scheduler. 10 úloh celkem.

TL;DR: Otevři DSM Control Panel → Task Scheduler a vytvoř 10 úloh dle této tabulky. Každá úloha je User-defined script, user root, command podle vzoru níže. Zaberou ti ~15 minut.

Společné nastavení (vzor)

Control Panel → Task Scheduler → Create → Scheduled Task → User-defined script
General: Task = název dle tabulky. User = root. Enabled = ✓
Schedule: dle sloupce „Spouštění" — viz „Schedule překlad" níže
Task Settings → Run command:
curl -fsS -X POST https://www.raseliniste.cz/api/cron/<ENDPOINT> \
     -H "x-cron-key: <CRON_SECRET>" \
     --max-time 120

Zaměň <ENDPOINT> a <CRON_SECRET>.

Kde najdu CRON_SECRET

SSH na NAS:

sudo grep CRON_SECRET /volume1/docker/raseliniste/.env

Nebo otevři ten soubor přes File Station / SFTP.

Schedule překlad

Co píšemeCo nastavit v DSM
každých 5 / 15 minRun on: Repeat, Frequency: Every 5 / 15 minutes
denně HH:MMRun on: Daily, First run time: HH:MM
poslední den měsíceRun on: Monthly, Date: Last day of month

10 úloh — checklist

1. Raseliniste sync calendars

každých 5 min · POST /api/cron/sync-calendars · max-time 120

K čemu: Stahuje Google Calendar primary + iCloud syn (RODINA) + iCloud partnerka (S & P). Klíčové pro to, aby kalendář v aplikaci byl aktuální (max 5 min pozadu za Fantasticalem). Bez něj nebudou fungovat ani booking sloty (klient by viděl jako volné to, co už máš obsazené ve Fantasticalu).

Test: v DSM pravý klik na task → Run → History → Result code 200. Plus zkontroluj v /calendar, že nový event z Fantasticalu se objeví do 5 min.

2. Raseliniste sync contacts

denně 04:00 · POST /api/cron/sync-contacts · max-time 120

K čemu: Stahuje Google People API (My Contacts) → Contact tabulku v Rašeliništi. Dedup s existujícími kontakty (vCard import + manuální + Google) podle e-mailu / telefonu. Délka noční sync, ne real-time — kontakty se nemění tak často.

Test: přidej nový kontakt v Google → druhý den v 04:01 už by měl být v /contacts.

3. Raseliniste nightly briefing

denně 22:00 · POST /api/cron/nightly-briefing · max-time 120

K čemu: Vygeneruje pro zítřek strukturovaný plán (schůzky, vzít s sebou, errands, kontextová varování, porušení pravidel) a pushne do Todoistu (projekt mojeUkoly) jako jeden task s due_string="today". Trvá typicky 30–60 s (Vertex Pro analyzuje).

Test: klikni Run kdykoliv → v Todoistu by se měl objevit task „🌙 Plán na zítra". Idempotent — 2× run = 1 task.

4. Raseliniste retry stuck recordings (Studna)

každých 15 min · POST /api/cron/retry-stuck-recordings · max-time 30

K čemu: Pickne Studna recordings ve stavu „processing" starší než 10 min — typicky uvázly při restartu kontejneru (fire-and-forget Promise umřela). Spustí AI processing znovu z uloženého audia. Bez něj by záznamy zůstaly „processing" navěky.

Test: ve Studně nahraj záznam, hned po uploadu DSM → Container Manager → restart kontejneru. Po 15 min by měl recording být buď processed nebo error (ne pořád processing).

5. Raseliniste cleanup audio (Studna)

denně 03:00 · POST /api/cron/cleanup-audio · max-time 120

K čemu: Maže Studna STANDARD audia starší 14 dní (briefy + pinned se nemažou). Šetří místo na disku NAS. Transkripty + analýzy zůstávají v DB.

Test: nahraj testovací STANDARD záznam, počkej 14 dní (nebo upravíme threshold v kódu pro test). Po cron běhu by audio (file na disku) zmizelo, recording v DB by měl audioPath: null.

6. Raseliniste daily projects digest (Studna)

denně 18:00 · POST /api/cron/daily-projects-digest · max-time 120

K čemu: Pro každý ProjectBox se includeInDigest=true vyrobí denní souhrn, co se v něm hosti za den nahráli, a pošle ti ho mailem. Přeskočí se pokud nic nepřibylo.

Test: v Studně otevři projekt → Nastavení → zapni „Zahrnout do denního souhrnu". Klikni Run → mail by měl dorazit (nebo skip pokud projekt nemá nové recordings).

7. Raseliniste monthly health report

poslední den měsíce 23:00 · POST /api/cron/monthly-health-report · max-time 180

K čemu: Vertex Pro analyzuje tvá zdravotní data za uplynulý měsíc (Health Auto Export → HealthMetric tabulka), vyrobí markdown report a pošle ti ho na notificationEmail (z Settings → E-mail reporty). Uloží se do /health historie.

Test: klik Run → měl by vzniknout HealthAnalysis s trigger=MONTHLY_AUTO + mail. Pokud nemáš HealthAuto Export aktivní, dostaneš report s „chybí data".

8. Raseliniste cleanup expired invites (Booking)

denně 01:00 · POST /api/cron/cleanup-expired-invites · max-time 30

K čemu: Označí BookingInvity ve stavu PENDING/VIEWED/RESERVED s validUntil < now jako EXPIRED. Bez něj by zůstaly v aktivním stavu navěky.

Test: vytvoř pozvánku s validity 1 den → druhý den po 01:00 by měla mít status EXPIRED v /calendar/invite.

9. Raseliniste cleanup task audio batches

denně 02:30 · POST /api/cron/cleanup-task-audio-batches · max-time 60

K čemu: Audio retention pro /ukoly/audio diktáty. Maže audio file z batchů starších 7 dní (transkript zůstává). Maže celé batche (committed/discarded) starší 30 dní. Stuck processing >30 min → status=error (po retries).

Test: nadiktuj úkoly, commit. Po 7 dnech by měl batch.audioPath být null. Po 30 dnech celý batch zmizet.

10. Raseliniste retry stuck task batches

každých 5 min · POST /api/cron/retry-stuck-task-batches · max-time 30

K čemu: Analogicky #4 ale pro /ukoly/audio. Pickne TaskAudioBatch ve „processing" starší 5 min (= kontejner restartl uprostřed AI), spustí znovu. Bez něj by diktáty mohly uvíznout.

Test: nadiktuj úkoly, hned po uploadu restart kontejneru. Po 5 min by měl batch být buď v review nebo error.

Po vytvoření všech 10 — kontrola

DSM → Task Scheduler → seznam má 10 řádků s prefix „Raseliniste"
Všechny mají sloupec Status = Enabled
U každé pravý klik → Run → History → Last run = OK / Result code 200
Pro #1, #4, #10 (vysoká frekvence) zkontroluj History po 1 hodině — měl by tam být běh à 5/15 min

Troubleshoot

SymptomŘešení
HTTP 503 CRON_NOT_CONFIGUREDCRON_SECRET není v .env kontejneru. Recreate: ~/deploy.sh.
HTTP 401 UNAUTHORIZEDŠpatný x-cron-key header — překlep, nebo se neshoduje s env.
HTTP 500 + dlouhý errorChyba v aplikaci. sudo docker compose logs app --tail 50 na NASu, najdi stack trace.
HTTP 200 ale nic se nestaloCron prošel ale data nemá co dělat (např. žádné stuck batches). Přečti response JSON pro detail.
Synology timeoutZvedni --max-time. Default jsem dal 120 s, briefingy můžou trvat víc.

Dodatek — env proměnné související

V /volume1/docker/raseliniste/.env musí být:

ProměnnáK čemu
CRON_SECRETSdílený secret pro x-cron-key auth. Generuj openssl rand -base64 32.
SESSION_SECRETCookie + AES klíč. ≥ 32 znaků.
DATABASE_URLPostgres URL.
APP_URLhttps://www.raseliniste.cz
VERTEX_PROJECT + VERTEX_LOCATION + GOOGLE_APPLICATION_CREDENTIALSVertex AI (preferováno).
GEMINI_API_KEYAI Studio fallback (pro audia > 14 MB v Vertex módu).
GOOGLE_CLIENT_ID + GOOGLE_CLIENT_SECRET + GOOGLE_REDIRECT_URIGoogle OAuth (Calendar + Contacts).
BOOKING_MAGIC_LINK_SECRETHMAC pro magic-link maily. Fallback: SESSION_SECRET.
BOOKING_MAGIC_LINK_TTL_HOURSDefault 24.
SIRI_API_TOKENPro budoucí Siri shortcut „zkontroluj termín" (volitelné).
RESEND_API_KEY nebo SMTP cfgPosílání mailů (Resend nebo Seznam SMTP).
NOTIFICATION_FROM + NOTIFICATION_EMAILMail odesílatel + globální fallback příjemce.
